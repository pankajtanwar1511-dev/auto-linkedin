TOPIC: Factory Pattern (Factory Method and Abstract Factory)

Creational design patterns for object creation without specifying exact classes

What this carousel covers:
→ Factory Method pattern for single product hierarchies with inheritance-based extension
→ Abstract Factory pattern for creating families of related objects without concrete classes
→ Decoupling client code from concrete implementations through interface-based creation
→ Static factories, registry-based factories, and template-based factory implementations
→ Exception safety in factory methods using smart pointers and RAII guarantees
→ Object slicing prevention when returning polymorphic objects from factories
→ Thread safety in lazy factory initialization with double-checked locking
→ Type erasure techniques using std::variant and std::any for multiple return types
→ Circular dependency resolution with abstract factories and two-phase initialization
→ Real-world autonomous vehicle examples with sensor/perception factories

Key insights:
◆ Factory Method delegates instantiation to subclasses for single product types, while Abstract Factory creates entire families of related objects
◆ Always return smart pointers (unique_ptr/shared_ptr) from factories to ensure exception safety and prevent resource leaks during initialization
◆ Returning polymorphic objects by value causes object slicing and breaks virtual dispatch - always use pointers or references
◆ Registry-based factories with std::map and lambda creators enable runtime plugin architectures without recompilation
◆ Function-local static (Meyers Singleton) prevents static initialization order fiasco in factory registries
◆ Template factories provide compile-time polymorphism with perfect forwarding for zero-overhead creation
◆ Factory pattern embodies Open/Closed Principle, Dependency Inversion, and Single Responsibility design principles
◆ Choose between inheritance (traditional polymorphism), std::variant (type-safe unions), or std::any (runtime type erasure) based on factory return type needs
