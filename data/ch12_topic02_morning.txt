TOPIC: Functor Pattern & Callable Objects

Classes with overloaded operator() that act like functions but preserve state

What this carousel covers:
→ Function objects (functors) via operator() overloading for stateful computation
→ State preservation across calls unlike function pointers
→ Compiler inlining advantages making functors faster than function pointers
→ Thread-safety patterns with std::atomic and std::mutex for concurrent access
→ Const-correctness with mutable keyword for caches and counters
→ STL algorithm copy semantics and using std::ref() to prevent state loss
→ Memoization with efficient single-lookup using find() instead of count() + []
→ Cache invalidation strategies when functor dependencies change
→ Custom predicates and comparators for STL containers (sort, find_if, priority_queue)
→ Generic functors with variadic templates and perfect forwarding

Key insights:
◆ Functors enable compiler inlining of operator() eliminating function call overhead, making them 10-15% faster than function pointers in STL algorithms
◆ STL algorithms copy functors by default - use std::ref() to pass by reference or capture the returned functor to preserve state modifications
◆ Avoid double hash lookups by using find() once instead of count() followed by [] operator in memoization patterns
◆ Mark operator() as const when it doesn't modify state; use mutable for caches, statistics, and lazy initialization that don't affect logical state
◆ Thread-safe functors require std::atomic for simple counters (lock-free) and std::mutex with minimal lock scopes for complex state like maps
◆ Cache invalidation is critical in memoizing functors - always clear cache when external dependencies change to prevent stale results
◆ RAII principles apply to resource-owning functors - use smart pointers, delete copy constructors, implement move semantics for file handles and connections
◆ Autonomous vehicle systems extensively use functors for sensor fusion (Kalman filtering), trajectory scoring comparators, and stateful PID controllers
