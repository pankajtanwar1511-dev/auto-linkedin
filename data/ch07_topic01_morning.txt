TOPIC: Template Fundamentals and Specialization

Master C++'s primary generic programming mechanism that enables type-safe, zero-overhead abstractions through compile-time polymorphism and code generation.

What this carousel covers:
→ Function vs class templates - automatic type deduction vs explicit instantiation
→ Template type deduction rules - strips top-level const, preserves low-level const
→ Two-phase name lookup - definition-time vs instantiation-time error checking
→ Full specialization - complete replacement for specific types (template<> class C<int>)
→ Partial specialization - pattern matching for type categories (T*, T&, T[N])
→ Non-type template parameters - compile-time constants (size_t N for array sizes)
→ Template metaprogramming - compile-time factorial/power via recursive instantiation
→ constexpr vs templates - modern value computation vs type-level computation
→ Real-world sensor fusion system with compile-time configuration

Key insights:
◆ Function templates deduce types automatically - template<typename T> T add(T a, T b)
◆ Class templates require explicit type specification - Box<int> b; (CTAD in C++17+)
◆ Type deduction strips top-level const/references - T param discards const int&
◆ Partial specialization enables type traits - TypeInfo<T*> matches any pointer type
◆ Most specific specialization wins - Full > Partial > Primary template priority
◆ Non-type parameters must be compile-time constants - constexpr values only
◆ Template metaprogramming is verbose - prefer constexpr functions for value computation
◆ Zero-overhead abstractions - templates inline completely, no runtime cost vs hand-coded
