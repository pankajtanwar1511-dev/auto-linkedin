TOPIC: Lock-Free Ring Buffer (Circular Queue)

Single Producer Single Consumer (SPSC) with atomic operations

What this carousel covers:
→ Fixed-size circular queue with atomic head/tail indices
→ Memory ordering: release-acquire semantics for visibility
→ False sharing prevention with 64-byte cache line alignment
→ Full vs empty disambiguation by sacrificing one buffer slot
→ Producer-consumer pattern with 1024 slots and 10,000 items
→ std::memory_order_relaxed, acquire, and release usage
→ Zero allocation and zero syscalls after initialization

Key insights:
◆ SPSC simplicity: Only one writer (no tail contention), one reader (no head contention) - eliminates CAS loops needed in MPMC
◆ Memory ordering: Producer uses memory_order_release on tail store, consumer uses memory_order_acquire on tail load for synchronization
◆ False sharing solution: alignas(64) on atomic head/tail puts them on separate cache lines, preventing ping-pong between CPU cores
◆ Full detection: next_index(tail) == head (sacrifices 1 slot) avoids ambiguity where head == tail could mean empty OR full
◆ Performance: Spin-waiting trades CPU cycles for latency - no context switches, no mutex syscalls, stays in userspace
◆ Real-world use: Audio/video pipelines, high-frequency trading, game engine render threads - where microseconds matter
