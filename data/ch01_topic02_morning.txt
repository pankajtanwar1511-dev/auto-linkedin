Key insights:
◆ Encapsulation is compile-time protection, not runtime security
◆ Virtual function calls in constructors/destructors always resolve to the current class's version, never to derived overrides
◆ Object slicing destroys polymorphism - always use references or pointers
◆ Virtual destructors are mandatory in polymorphic base classes
