📚 Today's Focus: std::set and std::map - Red-Black Tree Implementation

Key Concepts Covered:
• Red-Black tree structure with 5 balancing invariants (self-balancing BST)
• Guaranteed O(log N) performance for insert/find/erase operations
• Set stores unique keys; Map stores key-value pairs with const keys
• Memory overhead: ~32 bytes per element (3 pointers + color bit + data)
• Height bounded by 2 × log₂(N+1) with max 2 rotations per insertion
• Range query operations: lower_bound, upper_bound, equal_range (O(log N))
• Min/max access in O(1) via begin()/rbegin() iterators
• Iterator invalidation: only erased iterators invalidate, insertions safe
• Custom comparators for sorting (must maintain strict weak ordering)
• Map operator[] creates default value if key missing; insert() doesn't

Real-World Applications:
• Sorted dictionaries and symbol tables with predictable lookup times
• Range queries (timestamps, scores, prefix searches) with logarithmic complexity
• Real-time systems requiring guaranteed performance (no hash collision degradation)
• Frequency counting with sorted output (histogram generation)
• Membership testing with automatic deduplication and ordering
• Configuration storage with alphabetically sorted keys for readability

════════════════════════════════════════

💡 Note: Ordered containers use 800-900% more memory than vectors due to tree pointers, but provide O(log N) guaranteed performance vs unordered containers' O(1) average with O(N) worst case.
