TOPIC: C++11 Modern Initialization & Advanced Templates

Uniform initialization, initializer_list, variadic templates, and constexpr functions.

What this carousel covers:
→ Brace initialization syntax {} preventing narrowing conversions at compile-time
→ Most vexing parse resolution - braces vs parentheses for object declarations
→ std::initializer_list mechanics and constructor overload priority rules
→ Variadic templates with parameter packs and recursive expansion patterns
→ constexpr functions for compile-time computation with zero runtime cost
→ C++11 constexpr restrictions: single return statement and recursion-based logic
→ Perfect forwarding with variadic templates using std::forward
→ Template parameter pack expansion patterns and sizeof... operator
→ initializer_list lifetime issues and non-owning view semantics

Key insights:
◆ Brace initialization prevents narrowing (int x{3.14} is compile error, not silent truncation)
◆ initializer_list constructors always preferred with braces, even when other constructors match better
◆ Empty braces {} call default constructor, not initializer_list - use {{}} for empty list
◆ C++11 constexpr functions limited to single return with ternary operators - loops require recursion
◆ Variadic templates enable type-safe printf and perfect forwarding with arbitrary argument counts
◆ Parameter pack expansion syntax: args... forwards all, func(args)... applies function to each
◆ constexpr evaluation moves computation from runtime to compile-time: zero CPU cycles at runtime
◆ initializer_list is lightweight view (pointer + size), not owning container - lifetime dangers exist
