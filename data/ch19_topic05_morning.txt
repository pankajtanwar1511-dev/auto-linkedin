TOPIC: C++20 Standard Library Additions

New Tools for Modern Development

What this carousel covers:
→ std::span - Non-owning view (16 bytes on 64-bit: pointer + size)
→ std::format - Type-safe formatting (faster than iostreams, no virtual calls)
→ std::jthread - Auto-joining threads with stop_token cancellation
→ std::source_location - Automatic file/line/function capture without macros
→ Bit manipulation utilities - Hardware-accelerated operations (popcount, rotl/rotr)
→ Calendar and timezone support - Full IANA timezone database integration
→ Mathematical constants - std::numbers::pi, e, sqrt2, phi with type-specific versions
→ String enhancements - starts_with, ends_with for std::string and string_view
→ Synchronization primitives - std::latch (single-use), std::barrier (reusable), std::counting_semaphore
→ Utility functions - std::to_array, std::erase/erase_if, std::ssize, std::midpoint, std::lerp

Key insights:
◆ std::span provides unified interface for arrays/vectors/std::array - dynamic span is 16 bytes (pointer + size), fixed-extent span is 8 bytes (just pointer, size is template arg)
◆ std::format combines printf speed with iostream type-safety - compile-time format string validation, single allocation vs multiple for iostreams, comparable to snprintf performance
◆ std::jthread fixes std::thread safety - automatic joining in destructor (no std::terminate), built-in cooperative cancellation with stop_token/stop_source
◆ std::source_location eliminates macros - captures file_name, line, column, function_name at call site via default argument
◆ std::bit_cast ensures type-punning safety - compile-time size checks, requires trivially copyable types, no strict aliasing violations unlike reinterpret_cast
◆ Bit operations are hardware-accelerated - popcount counts 1-bits in single instruction, rotl/rotr use native CPU rotation, has_single_bit/bit_ceil/bit_floor for power-of-2 operations
◆ Timezone support with full DST handling - zoned_time converts between timezones, IANA database included, automatic daylight saving transitions
◆ std::latch vs std::barrier - latch for one-time countdown (cannot reuse), barrier for iterative synchronization (resets after all threads arrive, supports completion callback)
◆ std::counting_semaphore differs from mutex - allows N concurrent accesses (not just 1), no ownership concept (different threads can acquire/release), ideal for resource pools
◆ std::erase/erase_if improves erase-remove idiom - single function call instead of vec.erase(std::remove(...), vec.end()), works uniformly across all containers, returns count of erased elements
◆ std::midpoint provides overflow safety - computes (a+b)/2 without intermediate overflow even with INT_MAX values, works for integers and pointers
◆ std::format customization via std::formatter specialization - parse() handles format specs, format() outputs to context, enables custom types like Point to use std::format("{}", point)
