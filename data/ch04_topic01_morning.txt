TOPIC: Lvalue and Rvalue References

Understand value categories, reference binding rules, and the foundation of move semantics in modern C++.

What this carousel covers:
→ Lvalues vs rvalues - fundamental distinction between named objects and temporaries
→ Type categories vs value categories - separating static types from runtime value properties
→ Lvalue references (T&) - bind to persistent objects with identity and stable addresses
→ Rvalue references (T&&) - bind to temporaries and enable move semantics
→ Reference binding rules - const lvalue refs bind to both, rvalue refs only to temporaries
→ Named rvalue references are lvalues - having a name makes it an lvalue despite type
→ Universal references and forwarding references - T&& in template contexts
→ Common pitfalls - returning rvalue references to locals causes dangling references
→ Real-world autonomous vehicle example - efficient sensor data transfer patterns

Key insights:
◆ Having a name makes something an lvalue - even rvalue reference parameters are lvalues
◆ Rvalue references enable move semantics by identifying objects safe to "steal" from
◆ const lvalue references (const T&) are the "universal receiver" - bind to everything
◆ Returning rvalue reference to local variable creates dangling reference - always return by value
◆ Move semantics is built on distinguishing expiring objects (rvalues) from persistent ones (lvalues)
