TOPIC: C++11 Type Deduction (auto & decltype)

Automatic type inference that changes how you write modern C++

What this carousel covers:
→ auto keyword using template argument deduction rules with zero runtime overhead
→ Top-level const stripping vs low-level const preservation in auto deduction
→ Reference stripping behavior and when auto creates copies vs references
→ Array decay with auto vs auto& preserving array size information
→ Universal references with auto&& for perfect forwarding (lvalue/rvalue binding)
→ decltype exact type preservation including const, volatile, and reference qualifiers
→ decltype parentheses trap: decltype(x) vs decltype((x)) producing different types
→ Trailing return type syntax enabling type-dependent return values
→ Braced initializer special case deducing std::initializer_list in C++11/14
→ Performance implications in range-based for loops: auto vs const auto& vs auto&

Key insights:
◆ auto follows exact template argument deduction rules - strips top-level const and references by default
◆ Use const auto& in range-based loops to avoid expensive copies (30x speedup in real benchmarks)
◆ decltype preserves exact declared types including references - critical for template metaprogramming
◆ decltype((x)) with parentheses creates lvalue reference type vs decltype(x) returning declared type
◆ auto&& creates universal references with reference collapsing: lvalues→lvalue refs, rvalues→rvalue refs
◆ Plain auto in loops copies every element - use auto& for modification, const auto& for read-only
◆ Array decay: auto converts arrays to pointers, auto& preserves array type with size information
◆ Proxy types (std::vector<bool>) require auto&& or explicit types to avoid binding issues
◆ Trailing return type syntax (auto func() -> decltype(expr)) enables return type deduction in C++11
◆ auto with braced initializers deduces std::initializer_list in C++11/14 (changed in C++17)
