TOPIC: Advanced C++ Pitfalls - Compiler Optimizations, UB, and Modern C++ Traps

Subtle bugs that pass code review, work in debug builds, but fail catastrophically in production.

What this carousel covers:
→ How compilers exploit undefined behavior to remove safety checks and infinite loops
→ Null pointer checks removed after dereferencing due to UB assumptions
→ Signed integer overflow detection optimized away (vs. defined unsigned wrapping)
→ Array bounds violations transforming functions to always return true
→ Named rvalue references behaving as lvalues (requiring explicit std::move)
→ Const rvalues binding to copy constructors instead of move constructors
→ Perfect forwarding failures with braced initializers and bitfields
→ Forwarding references vs. rvalue references in class template contexts
→ C++20 constexpr vs. consteval vs. constinit - runtime vs. compile-time guarantees
→ Move semantics performance traps in autonomous vehicle sensor data processing

Key insights:
◆ Compilers optimize based on what the C++ standard allows, not programmer intent - UB assumptions enable aggressive transformations
◆ The triple threat: code passes review, works in debug (-O0), fails in production (-O2/-O3) when optimizations enabled
◆ Infinite loops with no observable side effects are UB and can be completely removed by optimizer
◆ Checking pointers AFTER dereferencing is too late - optimizer removes "unreachable" null checks
◆ Signed overflow checks like "if (x+y < x)" fail because signed overflow is UB - check BEFORE operations
◆ Named variables are always lvalues regardless of type - Widget&& rhs requires std::move(rhs.member) in move constructors
◆ Returning const prevents move optimization and RVO - always return non-const by value
◆ Template argument deduction ignores braced initializers - {1,2,3} has no type until context provides one
◆ Use AddressSanitizer (-fsanitize=address,undefined) to detect UB at runtime during development
◆ Real CVE examples: LibTIFF CVE-2019-14973 had overflow checks optimized away leading to security vulnerabilities
