TOPIC: RAII Fundamentals & Exception Safety

Resource Acquisition Is Initialization - the cornerstone principle of C++ resource management.

What this carousel covers:
→ RAII core principles: resources acquired in constructors, released in destructors
→ Resource lifetime bound to object lifetime with scope-based determinism
→ Stack unwinding mechanism and automatic cleanup during exception propagation
→ Exception safety guarantee levels: basic, strong, and nothrow guarantees
→ Deterministic destruction vs garbage collection timing and predictability
→ Partial construction edge cases and member initialization exception safety
→ Destructor exception dangers and the std::terminate() trap
→ Multiple resource management with strong exception safety patterns
→ Copy-and-swap idiom for transactional state modification guarantees
→ Rule of Three/Five/Zero and proper ownership semantics for RAII classes

Key insights:
◆ Stack unwinding automatically calls destructors for all constructed objects during exception propagation, making RAII the foundation of exception-safe code
◆ RAII eliminates entire categories of resource leaks by making cleanup structurally impossible to forget - destructors handle all exit paths automatically
◆ Member initializer lists provide strong exception safety: fully constructed members have destructors called even if later members throw during construction
◆ Never throw from destructors - if a destructor throws during stack unwinding while another exception propagates, std::terminate() is called and the program crashes
◆ Deterministic destruction provides compile-time guarantees and predictable cleanup timing at known code locations, unlike garbage collection's unpredictable delays
◆ Copy-and-swap idiom enables strong exception safety: state remains unchanged if operation fails, providing transactional all-or-nothing semantics
◆ Multiple resources require RAII wrappers for each member to ensure automatic rollback if any acquisition fails during construction
◆ RAII applies to ALL resources requiring explicit cleanup: memory, file handles, mutex locks, database connections, network sockets, and GPU resources
