Key insights:
◆ Struct and class are identical except default access - both support full OOP (inheritance, virtuals, constructors)
◆ Calling virtual functions from constructors always resolves to current class's version, never derived - causes UB for pure virtuals
◆ Follow Rule of Zero whenever possible using smart pointers - eliminates manual resource management bugs entirely
◆ Vtable and vptr enable runtime polymorphism - one pointer overhead per object, minimal cost for powerful flexibility
◆ Name hiding vs overriding - derived functions hide all base overloads unless using declaration restores them
◆ Explicit keyword prevents implicit conversions - mark single-parameter constructors explicit unless conversion intended
