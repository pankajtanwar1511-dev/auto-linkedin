TOPIC: Smart Pointers in Modern C++

Eliminate manual memory management with automatic RAII-based ownership semantics through unique_ptr, shared_ptr, and weak_ptr.

What this carousel covers:
→ Three smart pointer types - unique_ptr (exclusive), shared_ptr (shared), weak_ptr (non-owning)
→ Ownership models - single owner vs reference counted vs observer patterns
→ RAII automatic memory management - acquire in constructor, release in destructor
→ Performance characteristics - unique_ptr has zero overhead, shared_ptr uses atomic operations
→ Factory function patterns - returning smart pointers for clear ownership transfer
→ Function parameter guidelines - when to pass by value, reference, or raw pointer
→ Rule of Zero with smart pointers - compiler-generated special members work correctly
→ Edge cases - circular references with shared_ptr, polymorphic deletion, aliasing constructor
→ Real-world examples - autonomous vehicle trajectory management with proper ownership

Key insights:
◆ unique_ptr is the default choice - zero overhead with compile-time safety
◆ shared_ptr incurs atomic reference counting overhead - only use when truly sharing ownership
◆ Circular references with shared_ptr cause memory leaks - use weak_ptr to break cycles
◆ Always use make_unique/make_shared - single allocation is faster and exception-safe
◆ Virtual destructors are still required for polymorphic deletion with smart pointers
