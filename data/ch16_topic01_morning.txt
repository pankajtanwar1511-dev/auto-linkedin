TOPIC: C++17 Language Features

Three fundamental enhancements that make C++ more expressive and easier to use.

What this carousel covers:
→ Structured bindings for unpacking pairs, tuples, structs, and arrays with clean syntax
→ Copy vs reference semantics (auto, auto&, const auto&) with structured bindings
→ if constexpr for compile-time conditional branching in templates
→ How if constexpr discards dead branches and allows type-specific code
→ Replacing complex SFINAE patterns with readable if constexpr logic
→ Inline variables for header-only variable definitions without ODR violations
→ Combining inline and constexpr for safe header-only constants
→ Common pitfalls: lambda captures, temporary lifetime, and initialization order

Key insights:
◆ Structured bindings eliminate .first/.second boilerplate in map iteration with for (const auto& [key, value] : map)
◆ if constexpr completely discards non-taken branches before semantic analysis, enabling type-dependent code that wouldn't compile for all types
◆ Regular if compiles both branches even with compile-time conditions; if constexpr provides zero-overhead branch elimination
◆ SFINAE with enable_if requires multiple function overloads; if constexpr achieves the same with single function and clear branching
◆ Pre-C++17 static variables in headers created separate copies per TU causing linker errors; inline keyword merges definitions
◆ inline constexpr combines compile-time evaluation with header-only definition - best practice for header constants
◆ Structured bindings can't extend temporary lifetimes - const auto& with temporaries creates dangling references
◆ These three features work together: structured bindings for clean unpacking, if constexpr for type-based logic, inline variables for header-only config
