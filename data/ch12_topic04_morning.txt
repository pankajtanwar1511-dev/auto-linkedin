TOPIC: Object Pool & Memory Pool Patterns

Pre-allocate and manage reusable objects for deterministic performance in real-time systems.

What this carousel covers:
→ Object pool fundamentals: pre-allocation, free lists, and O(1) allocation/deallocation
→ Object pool vs memory pool: type-safe objects vs raw memory blocks
→ Real-time performance: 20x faster allocation (5ns vs 100ns) with deterministic timing
→ Autonomous vehicle applications: LiDAR point clouds, radar detections, particle filters
→ Thread-safety strategies: mutex locks, lock-free SPSC, and thread-local pools
→ Double-free detection using usage tracking arrays and validation
→ Expandable chunk-based pools with proper index calculations
→ Placement new and explicit destructors for non-trivial types
→ Alignment requirements for SIMD types (AVX, cache-line padding)
→ Pool exhaustion handling strategies and performance trade-offs

Key insights:
◆ Object pools eliminate memory fragmentation and provide zero-variance allocation timing (±1% vs ±10x for heap)
◆ Cache locality improves dramatically: 95% hit rate vs 60% with scattered heap allocations
◆ Critical for real-time systems: prevents GC pauses, meets hard deadlines (100Hz+ control loops)
◆ Index-based free lists use 4 bytes per entry vs 8 bytes for pointers (50% memory reduction on 64-bit)
◆ False sharing destroys performance: pad objects to cache-line boundaries (64 bytes) in multi-threaded pools
◆ For non-POD types, use placement new on allocation and explicit destructor calls on deallocation
◆ Lock-free SPSC pools achieve wait-free allocation using atomic head/tail indices with acquire/release ordering
◆ Chunk-based expansion enables dynamic growth while maintaining O(1) allocation through global index transformations
