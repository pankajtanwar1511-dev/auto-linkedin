TOPIC: Event-Driven Architecture (Asynchronous Systems)

Building reactive systems that communicate through events, not direct calls

What this carousel covers:
→ Core components: Events, Producers, Consumers, and Event Bus routing
→ Synchronous vs Asynchronous event handling with blocking behavior analysis
→ 4 event delivery patterns: Simple, Queued, Thread-Based, and Priority-Based
→ 6 critical edge cases: Iterator invalidation, event loops, race conditions, memory leaks, event storms, exception safety
→ Real-world implementations: GUI systems (Qt signals/slots), game engines, IoT sensor networks
→ Thread-safe event bus with mutex protection and lock optimization
→ Event patterns: Observer, Pub/Sub, Event Sourcing, Command Pattern

Key insights:
◆ Event-Driven = loose coupling where producers don't know consumers, enabling scalable many-to-many communication
◆ Synchronous events block until all handlers complete (easy debugging), Asynchronous events queue and return immediately (better performance)
◆ Iterator invalidation bug: Handler calls unsubscribe() during publish() crashes - fix by copying handler list before iteration
◆ Event loops cause infinite recursion - prevent with max depth limit (10 levels), deferred event queue, or self-source checks
◆ Race conditions in async events solved with sequence numbers, mutex locks for critical state, or synchronous critical operations
◆ Memory leaks from dangling handlers fixed with RAII Subscription handles, weak_ptr checks, or explicit unsubscribe in destructor
◆ Event storms (1000 events/sec) overwhelm system - mitigate with bounded queues (max 1000), rate limiting (10 events/sec), event coalescing (keep latest only), or sampling (process every 10th)
◆ Exception in one handler stops all others - wrap each handler in try-catch to ensure all handlers run even if some throw
◆ Priority event bus uses max heap to process high-priority events (value 10) before normal events (value 1) for safety-critical systems
◆ Thread-safe event bus copies handlers under lock then calls without lock to minimize contention in multi-threaded systems
