TOPIC: C++ Cast Operators

Replace dangerous C-style casts with type-safe C++ cast operators that make intent explicit and enable compile-time checking.

What this carousel covers:
→ Four C++ cast operators - static_cast, dynamic_cast, const_cast, reinterpret_cast
→ static_cast for related types - numeric conversions, inheritance upcasts, known downcasts
→ dynamic_cast with RTTI - runtime-checked polymorphic conversions, returns nullptr on failure
→ const_cast for const removal - only cast that adds/removes const/volatile qualifiers
→ reinterpret_cast for low-level operations - bitwise pointer reinterpretation, highly dangerous
→ C-style cast dangers - performs any cast possible, hides dangerous operations
→ Safety hierarchy - dynamic_cast (safest), static_cast (medium), const_cast (low), reinterpret_cast (lowest)
→ Runtime cost - dynamic_cast has RTTI lookup overhead, others are zero-cost
→ Real-world scenarios - safe downcasting, legacy API integration, hardware register access

Key insights:
◆ static_cast for compile-time checked conversions - safe upcasts, explicit numeric truncation
◆ dynamic_cast returns nullptr for pointers, throws bad_cast for references on failure
◆ const_cast is code smell - indicates const-correctness issue in design or legacy API
◆ reinterpret_cast is UB-prone - violates strict aliasing, only for platform-specific code
◆ Never use C-style casts - static_cast + const_cast silently, hides intent and dangers
