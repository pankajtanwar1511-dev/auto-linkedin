TOPIC: Type Conversions and Type Deduction

Master implicit/explicit conversions, auto type deduction rules, and decltype to write type-safe, maintainable modern C++ code.

What this carousel covers:
→ Implicit vs explicit conversions - compiler-automatic vs programmer-requested type changes
→ Type promotion vs conversion - safe widening (char→int) vs potential data loss
→ explicit keyword - prevents unwanted implicit conversions through constructors
→ auto type deduction rules - strips top-level const and references by default
→ Top-level vs low-level const - variable itself const vs what it points to is const
→ auto& preserves const and references - prevents accidental copies
→ auto&& (forwarding references) - binds to both lvalues and rvalues with perfect forwarding
→ decltype mechanics - preserves exact type including const, references, and value category
→ Real-world examples - preventing accidental conversions and ensuring correct deduction

Key insights:
◆ auto strips top-level const and references - auto x = const_ref creates mutable copy
◆ Low-level const is preserved - auto p = const_int_ptr keeps const int*, not int*
◆ Use auto& to avoid copies - auto& x = large_vector binds reference, no copy
◆ decltype preserves everything - decltype(expr) keeps const, references, value category
◆ C++ allows max ONE user-defined conversion in implicit sequence - prevents confusing chains
