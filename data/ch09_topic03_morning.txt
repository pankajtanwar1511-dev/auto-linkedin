TOPIC: C++11 Inheritance Control - override and final Keywords

Compile-time polymorphism safety and explicit inheritance control

What this carousel covers:
→ override keyword - compile-time verification of virtual function overriding
→ Catching silent polymorphism bugs: name typos, const mismatches, parameter differences
→ final keyword's dual purpose: sealing classes and preventing method overrides
→ Signature matching rules: parameters, const qualification, reference qualifiers
→ Covariant return types with override checking
→ Virtual destructor override patterns for proper cleanup
→ Combining override and final for leaf implementations
→ Private virtual functions and Non-Virtual Interface (NVI) idiom
→ Compiler optimizations enabled by final: devirtualization and inlining
→ Real-world patterns: sensor hierarchies, factory methods, template methods

Key insights:
◆ Before C++11: signature mismatches silently created new functions instead of overriding - bugs detected only at runtime
◆ override transforms 40-60% of polymorphism bugs into compile-time errors (typos, const/param mismatches)
◆ What override checks: function name, parameter types/count, const/volatile qualification, reference qualifiers (&/&&), return type compatibility
◆ final class prevents any inheritance; final method prevents further override while allowing class inheritance
◆ Combining override final: override once from base, then seal - prevents further derivation at that hierarchy point
◆ Zero runtime overhead: both are compile-time features; final enables devirtualization for performance gains
◆ Covariant returns allowed: override accepts Derived* return when base returns Base* (factory pattern)
◆ Access specifiers independent: private virtual functions can be overridden (NVI pattern) - override still enforces signature matching
◆ Exception specifications: can override with more restrictive spec (noexcept override of throwing function is valid)
◆ Production safety: override catches refactoring errors when base signatures change; final seals critical safety logic in certified implementations
