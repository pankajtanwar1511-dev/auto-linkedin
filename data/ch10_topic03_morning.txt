TOPIC: Smart Pointers as RAII

Modern memory management through scope-based destruction

What this carousel covers:
→ unique_ptr for exclusive ownership with zero overhead and move-only semantics
→ shared_ptr reference counting architecture with control blocks and atomic operations
→ weak_ptr for non-owning observation to break circular references
→ Custom deleters for managing FILE*, sockets, and non-memory resources
→ make_unique and make_shared exception safety and single-allocation optimization
→ Array specialization: unique_ptr<T[]> using delete[] vs delete
→ enable_shared_from_this pattern for safe shared_ptr<this> creation
→ Aliasing constructor for managing pointers to sub-objects
→ Thread safety details: atomic control blocks vs unsynchronized managed objects
→ Observer patterns and cache implementations with weak_ptr

Key insights:
◆ unique_ptr is move-only with same size as raw pointer (8 bytes), enforcing exclusive ownership at compile time with stateless deleters having zero overhead
◆ shared_ptr uses 16 bytes (2 pointers) with separate control block storing strong/weak ref counts, type-erased deleter, and allocator—all ref count ops are atomic
◆ make_shared allocates object + control block contiguously (1 allocation vs 2), improving cache locality, but object memory persists until all weak_ptrs destroyed
◆ Circular references with shared_ptr cause memory leaks—use weak_ptr for back-references in doubly-linked lists, tree parents, and observer patterns
◆ weak_ptr::lock() returns valid shared_ptr if object alive or empty shared_ptr if expired—safe dangling pointer prevention without ownership
◆ Custom deleters in unique_ptr are template parameters (affect type), while shared_ptr deleters are type-erased in control block (same type, no size overhead)
◆ enable_shared_from_this stores internal weak_ptr to control block, preventing double-free from creating multiple independent control blocks for same object
◆ shared_ptr control block operations are thread-safe (atomic), but accessing managed object requires external synchronization like any shared data
