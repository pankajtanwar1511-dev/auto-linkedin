TOPIC: C++20 Concepts & Constraints

Revolutionary Compile-Time Type Checking

What this carousel covers:
→ Named compile-time predicates that replace cryptic SFINAE patterns
→ 20+ standard concepts library (std::integral, std::floating_point, std::copyable, std::regular)
→ Four ways to apply constraints (requires clause, trailing requires, constrained parameter, abbreviated template)
→ Requires expressions with 4 requirement types (simple, type, compound, nested)
→ Constrained auto and abbreviated function templates
→ Concept composition with logical operators (&&, ||, !)
→ Subsumption rules for overload resolution
→ Direct comparison: SFINAE vs Concepts (50-200 line errors → 5-10 lines)

Key insights:
◆ Error message transformation: Pre-C++20 SFINAE generates 100+ lines of template instantiation errors, C++20 Concepts produce clear "constraint not satisfied" messages in 5-10 lines
◆ Standard concepts organized by category: Core language (same_as, derived_from, convertible_to), Numeric (integral, signed_integral, floating_point), Comparison (equality_comparable, totally_ordered), Objects (movable, copyable, semiregular, regular), Callable (invocable, predicate, relation)
◆ Subsumption determines overload resolution: SignedIntegral = Integral && IsSigned means SignedIntegral subsumes Integral, making it more constrained and preferred for signed types
◆ Four requirement types in requires-expressions: Simple (++x validity), Type (typename T::value_type existence), Compound ({x.size()} -> std::convertible_to<size_t> return type), Nested (requires sizeof(T) <= 8 compile-time boolean)
◆ Constrained auto enables type-safe deduction: std::integral auto x = getValue() replaces unconstrained auto with compile-time verification
◆ Abbreviated function templates simplify syntax: void func(std::integral auto x) equivalent to template<std::integral T> void func(T x)
◆ Concept composition with precedence: ! > && > ||, use parentheses for clarity in complex constraints
◆ Critical gotcha: requires requires is valid C++20 - outer starts requires-expression, inner checks compile-time boolean condition
◆ Compile-time performance improvement: Concepts enable early rejection before template instantiation, faster than SFINAE's substitution approach
◆ Migration advantage: Concepts are strictly better than SFINAE - no reason to use enable_if_t in C++20, purely legacy for C++17 compatibility
