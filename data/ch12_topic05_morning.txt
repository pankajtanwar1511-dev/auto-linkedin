Building a production-ready dynamic array from scratch.

What this carousel covers:
→ Three-pointer architecture (data, size, capacity) for dynamic growth
→ Geometric growth strategy: capacity doubling for amortized O(1) push_back
→ Rule of Five implementation: destructor, copy/move constructors, copy/move assignment
→ Iterator invalidation patterns and prevention strategies with reserve()
→ Copy-and-swap idiom for strong exception safety guarantees
→ Move semantics with moved-from state management (valid but unspecified)
→ Reserve vs resize semantics: capacity optimization vs element construction
→ RAII-based automatic memory management with proper cleanup
→ Contiguous memory layout for superior cache locality
→ Real-world LiDAR point cloud processing (100K-2M points per frame)

Key insights:
◆ Capacity doubling ensures O(1) amortized insertion despite O(N) reallocation cost
◆ Iterator invalidation occurs on reallocation: all pointers/iterators to freed memory become invalid
◆ Copy-and-swap prevents broken state: risky operations in temp object, noexcept swap on success
◆ Move constructors must reset source object (data=nullptr, size=0) to prevent double-delete
◆ reserve() allocates space without constructing elements; resize() changes size and constructs/destructs
◆ Strong exception guarantee: assignment operations are all-or-nothing via temporary copy
◆ Contiguous storage provides 10-100x better cache performance than linked structures
◆ Memory overhead: typical capacity is 1.5x-2x actual size for growth headroom
