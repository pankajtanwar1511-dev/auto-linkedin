TOPIC: Constructor Types - Default, Parameterized, Copy, and Move

Object lifecycle management and the Rule of Five for resource-safe C++ code.

What this carousel covers:
→ 4 constructor types - default, parameterized, copy, move with auto-generation rules
→ Member initialization methods - initializer list vs constructor body vs in-class defaults
→ CRITICAL initialization order bug - declaration order, NOT initializer list order causes UB
→ Built-in types contain garbage if uninitialized - source of heisenbugs
→ Const and reference members - MUST use initializer lists, assignment fails
→ Rule of Five - if you define one special member, define or delete all five
→ Move constructors - rvalue references, mark noexcept for container performance
→ Copy elision and RVO - C++17 mandatory elision eliminates copy/move
→ Delegating constructors - centralize init logic, reduce duplication
→ Explicit keyword - prevent surprising implicit conversions
→ Autonomous vehicle trajectory example - all constructor types with deep vs shallow copy

Key insights:
◆ Members initialize in declaration order, not list order - mismatch causes undefined behavior using uninitialized values
◆ Built-in types are garbage unless explicitly initialized
◆ Initializer lists are mandatory for const/ref and more efficient for all types (direct construction vs default-construct-then-assign)
◆ Declaring any constructor suppresses default constructor - use = default to restore
