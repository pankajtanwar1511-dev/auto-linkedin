TOPIC: Memory Management in C++

Master stack and heap memory allocation, understand RAII principles, and learn how smart pointers eliminate manual memory management bugs.

What this carousel covers:
→ Stack vs heap memory - automatic vs manual lifetime management, LIFO vs random access patterns
→ Stack allocation speed (~1-2 CPU cycles) vs heap allocation overhead (~100-1000+ cycles)
→ new/delete operators and array allocation/deallocation correctness
→ RAII (Resource Acquisition Is Initialization) - automatic resource cleanup via destructors
→ Smart pointers - unique_ptr for exclusive ownership, shared_ptr for reference counting
→ Common memory bugs - leaks, dangling pointers, double deletion, buffer overflows
→ Exception safety through RAII - guaranteed cleanup during stack unwinding
→ Rule of Zero - eliminating manual memory management by using RAII types
→ Real-world autonomous vehicle sensor data management with perception pipeline example

Key insights:
◆ Stack allocation is ~100-1000× faster than heap but limited to 1-8 MB
◆ RAII makes exception-safe code the default - destructors run automatically during unwinding
◆ Never mix malloc/free with new/delete - incompatible allocators cause undefined behavior
◆ Mismatched operators (new[] with delete or malloc with delete[]) corrupt heap allocator
◆ Always prefer std::make_unique and std::make_shared over raw new for exception safety
