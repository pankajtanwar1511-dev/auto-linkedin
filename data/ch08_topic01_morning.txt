TOPIC: std::vector - Dynamic Contiguous Array

Master the most frequently used STL container combining C array efficiency with automatic memory management through RAII and geometric growth strategies.

What this carousel covers:
→ Three-pointer architecture - begin_, end_, capacity_ pointers define size and capacity
→ Geometric growth strategy - 2x or 1.5x capacity growth for amortized O(1) push_back
→ Contiguous memory benefits - cache locality, SIMD vectorization, O(1) random access
→ Iterator invalidation rules - reallocation invalidates all, erase invalidates ≥pos
→ reserve() vs resize() - allocate space without construction vs create elements
→ Memory reclamation - clear() keeps capacity, swap idiom frees memory
→ Erase-remove idiom - O(n) removal vs O(n²) repeated erase
→ Vector vs list tradeoffs - cache performance dominates algorithmic complexity
→ Real-world measurements - iteration 10-100x faster than list despite same O(n)

Key insights:
◆ Size vs capacity distinction - size() is valid elements, capacity() is allocated space
◆ Accessing reserved but uninitialized elements is UB - reserve() doesn't create elements
◆ Reallocation triggers at size == capacity - use reserve() to prevent iterator invalidation
◆ push_back is O(1) amortized - occasional O(n) reallocation amortized across insertions
◆ Capacity never decreases automatically - use swap idiom or shrink_to_fit()
◆ Vector sorting 5-10x faster than list - cache locality beats O(n log n) complexity
◆ emplace_back avoids temporaries - constructs in-place using perfect forwarding
◆ Mark move constructors noexcept - enables move during reallocation for strong exception guarantee
