TOPIC: Perfect Forwarding and Reference Collapsing

Master universal references, reference collapsing rules, and std::forward to enable perfect argument forwarding in C++ templates.

What this carousel covers:
→ Universal references (forwarding references) - T&& where T is template-deduced parameter
→ Critical distinction - T&& is universal ONLY when type deduction occurs at call site
→ Reference collapsing rules - 4 combinations that enable universal references to work
→ Simple collapsing rule - any lvalue reference (&) in chain produces lvalue reference
→ Named rvalue reference paradox - T&& parameter type vs lvalue expression behavior
→ std::forward mechanism - conditional cast that preserves original value category
→ Perfect forwarding pattern - wrapper functions that forward arguments unchanged
→ Common pitfalls - adding const breaks universal references, class member T&& is NOT universal
→ Real-world factory pattern example with variadic template perfect forwarding

Key insights:
◆ T&& is universal reference ONLY when T is deduced - not with concrete types or fixed class templates
◆ Reference collapsing: & + anything = &, only && + && = && (lvalue reference wins)
◆ Named rvalue references are lvalues - parameter int&& x is an lvalue expression named "x"
◆ std::forward<T>(x) casts to T&& - preserves lvalue/rvalue-ness based on deduced T
◆ Without std::forward, all forwarded arguments become lvalues regardless of original category
